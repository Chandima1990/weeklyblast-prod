Setup the Environment

Node

Installing Node

The application runs with Node version 12.13.0

64 bit *.msi version can be downloaded here. If any other format is the required please go here and download the format.

Just go by the default installation criteria. When it asks for installing extra content such as python libraries, just make it a "NO"

Angular

Installing Angular

This application runs with angular version 8.3.23

To install angular as a global environment (Can install only in a folder by removing the '-g' tag. Installing globally makes your computer can run angular from any local location) just open a command prompt window as an administrator and enter the following command.

npm i -g @angular/cli@8.3.23

Press enter and wait until the installation is completed

Now you have installed angular as a global environment in your computer

Server

Installing Http-Server

This production build can be run on node http-server

To install http-server globally, again start an administrator command prompt window and enter following command.

npm i -g http-server

Press enter and wait until the installation is completed

Now you have installed http-server as a global node server in your computer.

Run

Run the Game

Now the game can be run on the http-server we installed previously.

Extract files in the WeeklyBlast zip to a folder. Open a command prompt window in that folder location. (Type cmd on folder path bar and press enter) Then enter the following command.

http-server --gzip --proxy http://localhost:8080?

Note: If the port is busy try any other port rather than 8080

Press enter and wait until the server starts. This will not take long, just a few seconds.

Now open a browser and enter following url

http://localhost:8080/

Setup the Game

Questions

Adding the questions

Questions can be added as a JSON file.

Open the WeeklyBlast folder we just extracted. navigate here.

src\assets\user_files

Find the .json file named game_list.json (This JSON can be stored anywhere and upload when running the application)

Open the JSON in a text editor

JSON

JSON explanation

Games can be conducted in two types. Each team answers to the question separately or open for any team to answer.

game_list
JSON tree structure
name (Mandatory)
name of the question shown in headers
operatingmode (Mandatory)
Specify the game operating type by "open" or "team"
rules (Optional)
Specify the rules as a comma separated string of the game per question, if any. otherwise leave an empty array.
subteamgame (Mandatory)
Specify the question and answer heading, if any. And question contents if any
team (Mandatory)

If the "operatingmode" was "open" then specify question or answer heading if any, otherwise just leave as "Question" and "Answer"

if "operatingmode" was team then specify the team names, if any. (Remember to have the same number of teams you are planning for the game)

image (Optional)
Question/Answer or team images as relevant, if any.
answer (Optional)
Answer by text.
content (Optional)
If the question has a video add this key with "videoplayback" as the value.
src (Conditional)
If "content" key has "videoplayback" as value specify the video file path stored in user files folder.
time (Mandatory)
Specify the time taken for the game by seconds

JSON example is already in the user files folder.

Host the Game

Team

Adding the teams

Navigate to the application which is already running in your browser.

Navigate to /teams by clicking on the link upper right corner.

Add the people available for the session on the right panel by clicking

After adding all the participants specify the number of teams you have planned for the game, as stated in the JSON.

Then click start. This will divide all in to teams randomly.

Then click download as *.csv and download the file to your computer

Now click the "Upload Team" blue color button and upload the *.csv file downloaded previously.

Now click the "Upload Game" blue color button and upload the *.json file prepared previously.

Navigate to /games by clicking "Start Battle" button or Games on the upper right corner.

Game

Hosting the game

Check the teams on the scoreboard on the left corner, first thing. If something is wrong with teams there will be a yellow refresh button on the top of the scoreboard. Click and refetch the teams. This button will be removed in 30 seconds if you did not take any action. If button is removed before you address it, just refresh the browser to take the button back to visible.

Everything is fine with the teams now you can perform the game as you planned.

To start the timer for a game click on

To stop the timer click on

To pause the timer click on

To restart the timer click on

If the sounds are too disturbing, you can always toggle the sounds by clicking on upper right corner

When someone gains a point the point can be added to that person on the scoreboard by clicking

When someone looses a point the point can be deducted from that person on the scoreboard by clicking

Each person's current score can be seen on there name as a badge and team score also can be seen near the team name.

When a team reaches the goal score usually they can be specified as won as number 1. But this can be customized as the game host's preference. So if the host is planning to deduct score even after a team won the 1st place, that is also possible.

Closing

Closing ceremony

Teams will be placed in positions as they scored in the game.

Places will be calculated by number of points acquired. If two teams are having the same score the first team, who came to that score will be prioritized.

Person of the series will be the person who has the most points acquired throughout the game.

Finally add the participants supposed to be in the next session and click spin for choosing next host for the game.
